# Acceptance Criteria

1. When a user opens a Google Calendar invite, the system should display an indicator showing their availability for the day of the meeting.
2. Availability should be calculated within the current day.
3. Only confirmed events should be counted as busy time.
4. The indicator should show the overall busy/free status for the entire day, not just the time slot of the invite.
5. If the user has back-to-back meetings with no gaps, the indicator should reflect this as "fully busy" without showing a specific warning.
6. The indicator should be read-only; users cannot take any action on their existing events from this view.
7. The feature should be available to all users without any special permissions or settings.
